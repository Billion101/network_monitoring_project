🤖 Solution 2 — Agent Push Mode

Complete Guide: Local Agent Push Architecture

Detailed step-by-step setup to poll Cisco SNMP locally inside the EVE-NG Linux VM and push telemetry metrics directly to the Cloud VPS over outbound HTTPS.

✓ Zero VPN Requirement ✓ Runs inside EVE-NG VM (Ubuntu) ✓ Outbound HTTP/HTTPS to VPS ✓ Auto-restarts with PM2
💡
Architecture Overview

Because your Cisco devices (192.168.100.x) sit inside EVE-NG's private internal virtual bridge, direct SNMP polling from the public internet (VPS) fails. In Agent Push Mode, a lightweight script (agent.js) runs locally on the EVE-NG Linux VM. It queries Cisco switches over local LAN (<1ms latency) and pushes JSON payloads to the VPS API endpoint POST /api/devices/ingest every 10 seconds.

🔄 Data Flow Comparison
❌ OLD IPsec VPN Way (Broken / Unreliable): VPS (72.62.76.1) ──→ IPsec VPN ──→ 192.168.100.2 (SNMP timeout) ✅ NEW Agent Push Way (100% Reliable): Cisco Switch (192.168.100.2) │ SNMP UDP 161 (Local Virtual Bridge) ▼ EVE-NG VM (192.168.43.253) ── [ agent.js ] │ HTTP POST /api/devices/ingest (Outbound connection) ▼ Cloud VPS Backend (72.62.76.1:5000) │ PostgreSQL status_logs + WebSocket Broadcast ▼ Dashboard UI (http://72.62.76.1:5173)

const snmp = require('net-snmp'); const http = require('http'); const VPS_HOST = '72.62.76.1'; const VPS_PORT = 5000; const POLL_MS = 10000; const COMMUNITY = 'darn@2026'; const DEVICES = [ { id: 2, name: 'Cisco_Firewall', ip: '192.168.100.1' }, { id: 3, name: 'Core_Switch', ip: '192.168.100.2' }, { id: 4, name: 'Access_Switch', ip: '192.168.10.252' }, ]; const OIDs = [ '1.3.6.1.4.1.9.2.1.56.0', // Cisco CPU 5-sec average % '1.3.6.1.4.1.9.2.1.8.0', // Free processor RAM '1.3.6.1.4.1.9.2.1.9.0', // Used processor RAM ]; function pollDevice(dev) { return new Promise((resolve) => { const session = snmp.createSession(dev.ip, COMMUNITY, { timeout: 3000, version: snmp.Version2c }); session.get(OIDs, (err, varbinds) => { session.close(); if (err) return resolve(null); const cpu = parseInt(varbinds[0]?.value) || 0; const free = parseInt(varbinds[1]?.value) || 1; const used = parseInt(varbinds[2]?.value) || 0; const mem = Math.round((used / (free + used)) * 100); resolve({ cpu, mem, status: 'online', trafficIn: 0, trafficOut: 0, latency: 1 }); }); }); } function postToVPS(path, body) { const data = JSON.stringify(body); const req = http.request({ hostname: VPS_HOST, port: VPS_PORT, path, method: 'POST', headers: { 'Content-Type': 'application/json', 'Content-Length': Buffer.byteLength(data) } }); req.on('error', (e) => console.error(`[PUSH ERROR] ${e.message}`)); req.write(data); req.end(); } async function runPollCycle() { console.log(`[CYCLE] Polling ${DEVICES.length} devices...`); for (const dev of DEVICES) { const m = await pollDevice(dev); const payload = m || { cpu: 0, mem: 0, status: 'offline', trafficIn: 0, trafficOut: 0, latency: 0 }; postToVPS('/api/devices/ingest', { deviceId: dev.id, ...payload }); if (m) console.log(` ✓ ${dev.name}: CPU ${m.cpu}%, MEM ${m.mem}%`); else console.log(` ✗ ${dev.name}: OFFLINE`); } } setInterval(runPollCycle, POLL_MS); runPollCycle(); console.log('[AGENT STARTED] Pushing SNMP metrics to VPS...');
