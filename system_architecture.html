1. High-Level Network & System Architecture

3. IPsec VPN Tunnel Infrastructure Setup

The IPsec VPN tunnel routes private lab subnets (192.168.0.0/16) directly to the Ubuntu VPS (72.62.76.1/32).

VPS Configuration: /etc/ipsec.conf (strongSwan)
🌐 VPS Server
config setup charondebug="ike 2, knl 2, cfg 2" uniqueids=no conn vps-to-eve-lab authby=secret auto=add # Wait as responder for incoming IKEv2 connection keyexchange=ikev2 left=72.62.76.1 # Public VPS IP leftsubnet=72.62.76.1/32 right=%any # Accepts EVE-NG NAT/WAN IP rightid=%any rightsubnet=192.168.0.0/16 # Route private lab subnets across tunnel ike=aes256-sha256-modp2048 esp=aes256-sha256
Cisco Firewall CLI Configuration (EVE-NG)
🔌 Cisco Device

4. Real-Time SNMP Telemetry Polling Engine

The backend service snmpService.js runs inside the netmonitor_backend Docker container. Every 5 seconds, it queries target Cisco devices across the active IPsec VPN tunnel using SNMP v2c.

backend/services/snmpService.js — Target Cisco OIDs
🌐 VPS Backend
// Target OIDs queried via SNMP v2c: const oids = [ '1.3.6.1.2.1.1.5.0', // sysName: Device Hostname (e.g. Core_switch) '1.3.6.1.4.1.9.2.1.56.0', // cpmCPUTotal5sec: Cisco 5-second CPU utilization (%) '1.3.6.1.4.1.9.2.1.8.0', // ciscoMemoryPoolFree: Free RAM (bytes) '1.3.6.1.4.1.9.2.1.9.0' // ciscoMemoryPoolUsed: Used RAM (bytes) ]; // Calculation formula for Memory Usage %: let mem = 0; if (usedRam + freeRam > 0) { mem = Math.round((usedRam / (usedRam + freeRam)) * 100); }
⚙️
Strict Real Mode (`ENABLE_SIMULATOR=false`): If SNMP polling fails or times out, the backend marks the device status as offline and records 0% CPU/Memory utilization in PostgreSQL.

5. Syslog Event Receiver (UDP Port 514)

The service syslogService.js listens on UDP 514 on the VPS. It processes incoming Cisco syslog RFC 5424 / RFC 3164 messages in real time (0s latency).

backend/services/syslogService.js — Ingestion Pipeline
🌐 VPS Backend
1. Listener: UDP 514 Socket binds to process incoming logs 2. Header Parsing: Extracts priority header `` facNum = Math.floor(priVal / 8); /* Facility e.g. local0, daemon */ sevNum = priVal % 8; /* Severity e.g. info, warning, err */ 3. Device Identification: Maps rinfo.address sender IP to database dev_id 4. Persistence: Executes `INSERT INTO syslogs (dev_id, facility, severity, log_message)` 5. Instant Broadcast: Pushes WebSocket event `{ type: 'syslog', data: { syslog, device } }`

8. Frontend UI Rendering & State Architecture (`frontend/src/App.tsx`)

The React SPA application is built with Vite, TypeScript, and TailwindCSS:

  • Dynamic Host Resolution: Automatically resolves API base to http://72.62.76.1:5000/api and WS base to ws://72.62.76.1:5000.
  • Session Persistence: User authentication state is stored in localStorage.netmonitor_authenticated, preserving session across browser refreshes.
  • Live Component Updates:
    • Topology Map Card View: Glowing SVGs change color dynamically (Green = Online, Yellow = Warning, Red = Offline).
    • Telemetry Gauges: Dynamic CPU and RAM utilization progress bars update every 5s.
    • Syslog Terminal View: Renders incoming syslogs with severity color coding (Emergency/Crit = Red, Warning = Yellow, Info = Blue).
