NetMonitor Console

Project Specification Overview

1. Project Goal & Core Objective

The primary goal of **NetMonitor** is to provide network administrators and NOC (Network Operations Center) engineers with a unified, real-time dashboard to monitor, track, and diagnose network devices hosted inside virtual lab environments (such as **EVE-ng**) or physical corporate topologies.

Real-Time State Mapping

Eliminates traditional dashboard delays by streaming metrics pings, CPU load, and bandwidth volumes directly to client screens using persistent WebSocket connections.

Dynamic Topology Trees

Renders complex connection paths from Internet Gateways, Perimeter Firewalls, Core Aggregators, and Access layer PCs into a single interactive vector diagram.

Proactive Incident Warnings

Collects raw console syslogs and triggers instant notifications when hardware nodes drop connection, exceed load baselines, or generate critical warnings.

2. System & Communication Architecture

The application is architected with a decoupled **SPA Frontend** and an **MVC Backend API** that synchronize via two separate communication pipelines:

Pipeline A: REST API (Request-Response)

Used for state transactions, credential verification, and loading static historical device records for charting purposes.

Pipeline B: WebSocket Stream (Real-Time Push)

Used to broadcast dynamic metrics and alert logs to all open browser windows as soon as updates are recorded in the system.

3. Technical Stack Implementations

Frontend Architecture
  • React 19 & TypeScript: Provides type-safe component state management.
  • Tailwind CSS v4: Curated dark-theme layouts featuring glassmorphism and subtle custom glowing animations.
  • SVG Vector Mapping: Custom layouts to render the tree connections between nodes dynamically.
  • Dynamic IP Resolution: Resolves backend URLs dynamically relative to the browser client location, ensuring zero manual config adjustments on VPS hosts.
Backend Architecture
  • Express Server Framework: Separates business logics into individual models, routes, and controllers (MVC pattern).
  • WebSockets Server (`ws`): Instantiates real-time listener handlers to broadcast metrics dynamically on server startup.
  • pg Connection Pool: Leverages connection caching limits to efficiently query relational databases without bottlenecking.
  • Simulations Worker: Simulates active corporate network hardware loads (ping, bandwidth, CPU) every 5 seconds.
